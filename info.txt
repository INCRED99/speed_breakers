Build a **modern, production-ready web application using CesiumJS** for visualizing and processing KML files related to road infrastructure.

---

# 🎯 Application Goal

Create a **GIS web tool** where users can:

1. Upload a `.kml` file containing placemarks on roads
2. Automatically process those placemarks (backend API will handle logic)
3. Visualize results on a **3D Cesium globe**
4. Display generated geometries such as:

   * road-aligned lines
   * rectangles (speed breakers / markings)
5. Download the processed KML file

---

# 🧱 Tech Stack

Frontend:

* React (with Vite)
* CesiumJS (latest version)
* TypeScript
* TailwindCSS

Optional:

* Zustand or Context API for state management
* Axios for API calls

# 🧭 Core Features

## 1. Cesium Viewer

* Full-screen Cesium globe
* Default camera focused on India (or user location)
* Enable:

  * zoom
  * pan
  * rotate
* Add base imagery (Bing or Cesium Ion)

---

## 2. KML Upload

* Drag-and-drop upload area
* Accept only `.kml`
* Show uploaded file name
* Parse KML and display placemarks on Cesium map

---

## 3. Visualization

After upload:

* Plot placemarks as:

  * colored points (blue)
* When processed:

  * show generated lines (red)
  * show polygons (yellow transparent)
* Smooth rendering and updates

---

## 4. Processing Button

Button:
**"Generate Road Geometry"**

On click:

* Send KML file to backend API
* Endpoint:

```id="api_endpoint"
POST /process-kml
```

* Show loading spinner

---

## 5. Display Results

After API response:

* Render:

  * snapped points
  * aligned lines
  * polygons
* Zoom camera to processed region

## 6. Download Output

Button:

**"Download Processed KML"**

* Trigger download from API response

## 10. Cesium Integration Details

* Use `Viewer` from Cesium
* Load KML using:

```id="cesium_kml"
Cesium.KmlDataSource.load()
```

* Add entities dynamically:

  * `viewer.entities.add()`
* Use:

  * `PolylineGraphics` for lines
  * `PolygonGraphics` for polygons
